## Reading Blocks and Control Flow from Java

In Musi, blocks and matches are expressions when they produce a value; the branch answer matters more than the statement container. For a Java reader, the key topic is blocks and control flow.

## Common mistake

Do not keep extra variables only to move a branch result out of a statement block. In Java, this appears often in blocks and control flow.

## When this pays off

Use this pattern when a route fee, access decision, or small rule table chooses one value from several cases. The Java habit still helps here: Keep the Java habit of naming APIs for future readers.
